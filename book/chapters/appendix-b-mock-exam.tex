\chapter{Mock Exam --- 40 Questions}
\label{app:mock}

Take this one under exam conditions. Phone in another room, single tab open, a timer in your eyeline. The forty questions below are calibrated to the C1000-185 blueprint and the question count is proportional to each section's weight, so the section that bites in production is the section that takes most of the test here too. Allow yourself sixty minutes --- the real exam is sixty-two questions in ninety, so the per-question time budget matches once you scale for question count. Flag the ones you are not sure about as you go; the second pass at the end is what catches the careless wrongs. The answer key, with chapter cross-references for any question you missed, is in Appendix~\ref{app:answers}.

\begin{enumerate}[leftmargin=*]

% ---------- Section 1 (15%) -----------
\item A team needs to tag every incoming support email with one of \texttt{BILLING}, \texttt{TECH}, \texttt{OTHER}. Which capability?
\begin{enumerate}[label=\Alph*.]
  \item Generation \item Summarization \item Classification \item Translation
\end{enumerate}

\item A 47B-parameter model routes each token to only $\sim$13B of its weights. Which pattern?
\begin{enumerate}[label=\Alph*.]
  \item Reasoning model \item Mixture of Experts \item VAE \item Encoder--decoder
\end{enumerate}

\item Internal handbook updated weekly. Best mitigation for outdated answers?
\begin{enumerate}[label=\Alph*.]
  \item Raise temperature \item Switch to 70B \item Add RAG over the handbook \item Reduce max\_new\_tokens
\end{enumerate}

\item Cost-conscious enterprise English FAQ bot, IBM-native preference. Best model?
\begin{enumerate}[label=\Alph*.]
  \item \texttt{meta-llama/llama-3-3-70b-instruct}
  \item \texttt{ibm/granite-3-2-8b-instruct} + RAG with \texttt{ibm/slate-125m-english-rtrvr-v2}
  \item Custom 13B from scratch \item \texttt{flan-t5-xxl} fine-tuned weekly
\end{enumerate}

\item A workflow needs to look up a contract, check inventory, decide on a refund, send email. Which architecture?
\begin{enumerate}[label=\Alph*.]
  \item Single prompt to a 70B model \item Plain RAG
  \item Agentic architecture with tools \item Fine-tuned classifier
\end{enumerate}

\item Which IBM-side family detects jailbreaks, HAP, and ungrounded RAG answers?
\begin{enumerate}[label=\Alph*.]
  \item watsonx.data \item Granite Guardian \item Tuning Studio \item Watson Assistant
\end{enumerate}

% ---------- Section 2 (16%) -----------
\item You need consistent, factual, reproducible output for an extraction task. Best decoding method?
\begin{enumerate}[label=\Alph*.]
  \item \texttt{sample}, temperature=1.0 \item \texttt{greedy}
  \item \texttt{sample}, top\_p=0.99 \item Beam search
\end{enumerate}

\item Model keeps repeating the same phrase. \emph{Single} parameter change with biggest payoff?
\begin{enumerate}[label=\Alph*.]
  \item Lower temperature to 0 \item Increase max\_new\_tokens
  \item Raise repetition\_penalty to $\sim$1.15 \item Switch to a smaller model
\end{enumerate}

\item Cap each call at 20s and 300 tokens. Which params?
\begin{enumerate}[label=\Alph*.]
  \item \texttt{stop\_sequences=["20s"]}
  \item \texttt{max\_new\_tokens=300, time\_limit=20000}
  \item \texttt{min\_new\_tokens=300, temperature=20}
  \item \texttt{repetition\_penalty=20}
\end{enumerate}

\item Versioning, governance, model-swap without app changes. Ship what?
\begin{enumerate}[label=\Alph*.]
  \item f-string in microservice \item \texttt{PromptTemplate} in a Deployment Space
  \item YAML file in repo \item Jinja on web server
\end{enumerate}

\item A power user wants to hand-craft a prompt with custom delimiters in watsonx Prompt Lab. Which mode?
\begin{enumerate}[label=\Alph*.]
  \item Chat \item Structured \item Freeform \item JSON-only
\end{enumerate}

\item The classifier output uses inconsistent label vocabulary across calls. Cheapest fix?
\begin{enumerate}[label=\Alph*.]
  \item Add few-shot examples and \texttt{stop\_sequences}
  \item Switch decoding to sample at temperature 1.5
  \item Use a different IBM region \item Train a new model from scratch
\end{enumerate}

\item Confident, fabricated case-law citation in output. Risk category?
\begin{enumerate}[label=\Alph*.]
  \item Prompt injection \item Hallucination \item PII leak \item Bias
\end{enumerate}

\item Hard prompts are preferred over soft prompts when the auditor needs to:
\begin{enumerate}[label=\Alph*.]

\item Run a 13B model on a single 24~GB GPU, still fine-tuning it. Best choice?
\begin{enumerate}[label=\Alph*.]
  \item Full FP32 fine-tune \item INT8 PTQ only
  \item QLoRA (4-bit base + LoRA adapters) \item BF16 inference, no tuning

\item Which two parameters constitute the core LoRA configuration?
\begin{enumerate}[label=\Alph*.]
  \item temperature, top\_p \item rank ($r$), alpha
  \item batch\_size, epochs \item privacy budget, $\delta$
\end{enumerate}

\item Which watsonx tool profiles a dataset and saves a repeatable cleaning flow?
\begin{enumerate}[label=\Alph*.]
  \item Prompt Lab \item Tuning Studio \item Data Refinery \item Watson Assistant
\end{enumerate}

\item Teach Granite a brand-new domain (new facts + new house style) at scale. Best method?
\begin{enumerate}[label=\Alph*.]
  \item Few-shot prompting \item Static prompt template
  \item InstructLab (taxonomy + SDG + multi-phase tuning) \item Pre-train from scratch
\end{enumerate}

\item Which test best validates that the \emph{tail} of a synthetic distribution matches the real one?
\begin{enumerate}[label=\Alph*.]
  \item Kolmogorov--Smirnov \item Anderson--Darling
  \item chi-square \item t-test
\end{enumerate}

\item Which DP parameter controls the \emph{probability} that the privacy guarantee fails?
\begin{enumerate}[label=\Alph*.]
  \item privacy budget ($\varepsilon$) \item leakage probability ($\delta$)
  \item random seed \item learning rate
\end{enumerate}

\item LoRA run with $r=8$, $\alpha=16$ overfits after epoch 2. Best first action?
\begin{enumerate}[label=\Alph*.]
  \item Increase $r$ to 64 \item Add LoRA dropout, reduce epochs
  \item Switch to full fine-tune \item Increase learning rate
\end{enumerate}

\item Lowest-cost legitimate fine-tuning option on watsonx for a small ($\sim$300 example) dataset?
\begin{enumerate}[label=\Alph*.]
  \item Full fine-tune \item Prompt tuning (soft prompt)
  \item LoRA $r$=128 \item Pre-training
\end{enumerate}

\item A teacher LLM generates thousands of training examples from a handful of human seeds, gated by a critic. What is this called in IBM's stack?
\begin{enumerate}[label=\Alph*.]
  \item AutoRAG \item LAB / InstructLab synthetic data generation
  \item Watson Discovery \item Watson Assistant
\end{enumerate}
