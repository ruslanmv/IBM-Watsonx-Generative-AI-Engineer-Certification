\section{Introduction to Generative AI}

\subsection{What is Generative AI?}

\begin{keybox}
\textbf{Generative AI} refers to artificial intelligence systems that can create new content—text, images, code, audio, or video—based on patterns learned from training data. Unlike traditional AI that classifies or predicts, generative AI \textit{generates} novel outputs.
\end{keybox}

Generative AI systems, particularly Large Language Models (LLMs), have transformed how we interact with technology. These models are trained on vast amounts of data and can:

\begin{itemize}[leftmargin=*]
    \item Generate human-like text responses
    \item Write and explain code
    \item Summarize and analyze documents
    \item Answer questions based on context
    \item Translate between languages
    \item Create structured data from unstructured input
\end{itemize}

\subsection{Why It Matters for the Exam}

Understanding the fundamental capabilities and limitations of generative AI is crucial because:

\begin{itemize}[leftmargin=*]
    \item \textbf{Design Decisions}: You must choose the right model and approach for specific use cases
    \item \textbf{Architecture Patterns}: Different problems require different architectural solutions
    \item \textbf{Risk Management}: Knowing limitations helps you implement appropriate safeguards
    \item \textbf{watsonx Integration}: IBM's platform is built around these core concepts
\end{itemize}

\section{The Five Core Capabilities of LLMs}

Large Language Models possess five fundamental capabilities that define their utility in enterprise applications:

\subsection{1. Text Generation and Completion}

\textbf{Concept}: The ability to generate coherent, contextually appropriate text based on a prompt or seed text.

\textbf{Applications}:
\begin{itemize}[leftmargin=*]
    \item Content creation (articles, marketing copy, documentation)
    \item Email composition and response generation
    \item Report writing and summarization
    \item Code generation and documentation
\end{itemize}

\subsection{2. Question Answering}

\textbf{Concept}: The ability to answer questions based on provided context or general knowledge.

\textbf{Types}:
\begin{itemize}[leftmargin=*]
    \item \textbf{Extractive QA}: Pulling exact answers from source documents
    \item \textbf{Abstractive QA}: Generating synthesized answers from multiple sources
    \item \textbf{Open-domain QA}: Answering based on model's training data
    \item \textbf{Closed-domain QA}: Answering within a specific knowledge domain
\end{itemize}

    \item Enabling retrieval-augmented generation for enterprise data
\end{enumerate}

\subsection{3. Summarization and Information Extraction}

\textbf{Concept}: The ability to distill large amounts of text into concise summaries while preserving key information.

\textbf{Summarization Types}:
\begin{itemize}[leftmargin=*]
    \item \textbf{Extractive}: Selecting key sentences from the original text
    \item \textbf{Abstractive}: Generating new sentences that capture the essence
    \item \textbf{Multi-document}: Synthesizing information from multiple sources
\end{itemize}

\textbf{Information Extraction Applications}:
\begin{itemize}[leftmargin=*]
    \item Named Entity Recognition (NER)
    \item Relationship extraction
    \item Event detection
    \item Structured data extraction from unstructured text
\end{itemize}

\subsection{4. Translation and Transformation}

\textbf{Concept}: The ability to convert text from one form to another while preserving meaning.

\textbf{Types of Transformation}:
\begin{itemize}[leftmargin=*]
    \item \textbf{Language Translation}: Converting between natural languages
    \item \textbf{Format Translation}: Converting between code languages (Python $\leftrightarrow$ JavaScript)
    \item \textbf{Style Transfer}: Changing tone (formal $\leftrightarrow$ casual)
    \item \textbf{Data Format Conversion}: JSON $\leftrightarrow$ XML $\leftrightarrow$ CSV
\end{itemize}

\section{Limitations of Generative AI and LLMs}

Understanding limitations is as important as understanding capabilities. The exam tests your ability to identify situations where generative AI may not be appropriate.

\subsection{Knowledge Cutoff and Outdated Information}

\begin{warningbox}
LLMs are trained on data up to a specific date (the \textbf{knowledge cutoff}). They cannot access real-time information or know about events after their training data was collected.
\end{warningbox}

\textbf{Implications}:
\begin{itemize}[leftmargin=*]
    \item Cannot provide current stock prices, weather, or news
    \item May give outdated technical recommendations
    \item Unaware of recent product launches or policy changes
\end{itemize}

\textbf{Solutions}:
\begin{itemize}[leftmargin=*]
    \item Implement Retrieval-Augmented Generation (RAG) for current data
    \item Use API calls to retrieve real-time information
    \item Regularly update fine-tuned models with new data
    \item Clearly communicate the knowledge cutoff to users
\end{itemize}

\subsection{Hallucinations and Factual Errors}

\textbf{Definition}: \textit{Hallucinations} occur when an LLM generates plausible-sounding but factually incorrect or fabricated information.

\textbf{Why Hallucinations Happen}:
\begin{itemize}[leftmargin=*]
    \item Probabilistic nature of text generation
    \item Training data may contain errors or biases
    \item Model "fills in gaps" when uncertain
    \item No inherent fact-checking mechanism
\end{itemize}

\examtip{The exam may present scenarios where you must identify hallucination risks and recommend mitigation strategies such as RAG, fact-checking layers, or confidence scoring.}

\textbf{Mitigation Strategies}:
\begin{enumerate}[leftmargin=*]
    \item \textbf{Retrieval-Augmented Generation}: Ground responses in verified documents
    \item \textbf{Temperature Control}: Lower temperature for factual tasks
    \item \textbf{Prompt Engineering}: Instruct the model to acknowledge uncertainty
    \item \textbf{Human-in-the-Loop}: Review high-stakes outputs
    \item \textbf{Confidence Scoring}: Implement mechanisms to detect uncertain responses
\end{enumerate}

\subsection{Context Window Limitations}

\textbf{Concept}: LLMs can only process a limited amount of text at once, called the \textit{context window} or \textit{token limit}.

\begin{table}[h]
\centering
\begin{tabularx}{\textwidth}{l c X}
\toprule
\textbf{Model} & \textbf{Context Window} & \textbf{Equivalent Pages} \\
\midrule
GPT-3.5 & 4,096 tokens & $\sim$3 pages \\
GPT-4 & 8,192-32,768 tokens & $\sim$6-25 pages \\
Llama-2 70B & 4,096 tokens & $\sim$3 pages \\
Claude 2 & 100,000 tokens & $\sim$75 pages \\
\bottomrule
\end{tabularx}
\caption{Typical Context Window Sizes (1 token $\approx$ 0.75 words)}
\label{tab:context-windows}
\end{table}

\textbf{Implications}:
\begin{itemize}[leftmargin=*]
    \item Cannot process entire large documents in one pass
    \item Long conversations may lose early context
    \item Trade-off between context size and computational cost
\end{itemize}

\textbf{Workarounds}:
\begin{itemize}[leftmargin=*]
    \item Chunking and processing in segments
    \item Summarization of earlier context
    \item Using vector databases for semantic search
    \item Implementing RAG for large document collections
\end{itemize}

\subsection{Lack of Common Sense and Real-World Understanding}

LLMs lack true understanding of the physical world, causality, and common sense reasoning.

\textbf{Examples of Common Sense Failures}:
\begin{itemize}[leftmargin=*]
    \item Cannot reliably answer "If I drop a glass, will it float or fall?"
    \item May give logically inconsistent responses
    \item Lacks understanding of physical constraints
    \item Cannot perform multi-step mathematical reasoning reliably
\end{itemize}

\subsection{Bias and Fairness Issues}

LLMs can perpetuate and amplify biases present in their training data.

\textbf{Types of Bias}:
\begin{itemize}[leftmargin=*]
    \item \textbf{Gender Bias}: Stereotyping occupations or characteristics
    \item \textbf{Cultural Bias}: Favoring certain cultural perspectives
    \item \textbf{Racial Bias}: Discriminatory associations or stereotypes
    \item \textbf{Socioeconomic Bias}: Assumptions based on economic status
\end{itemize}

\section{Architecture Patterns for Generative AI Solutions}

\subsection{Pattern 1: Prompt-Based Inference}

The simplest pattern: send a prompt, receive a response.

\textbf{Use When}:
\begin{itemize}[leftmargin=*]
    \item Task is within model's training data
    \item No specialized knowledge required
    \item Low latency is important
    \item Quick prototyping
\end{itemize}

\subsection{Pattern 2: Retrieval-Augmented Generation (RAG)}

Combines LLMs with external knowledge retrieval.

\textbf{Use When}:
\begin{itemize}[leftmargin=*]
    \item Need access to current or proprietary information
    \item Domain-specific knowledge required
    \item Want to ground responses in verified sources
    \item Reducing hallucinations is critical
\end{itemize}

\subsection{Pattern 3: Fine-Tuned Model}

Adapting a pre-trained model for specific tasks or domains.

\textbf{Use When}:
\begin{itemize}[leftmargin=*]
    \item Domain-specific language or terminology
    \item Consistent formatting required
    \item Prompt engineering alone is insufficient
    \item Have sufficient training data
\end{itemize}

\subsection{Pattern 4: Agent-Based Systems}

LLMs that can use tools, make decisions, and execute multi-step workflows.

\textbf{Use When}:
\begin{itemize}[leftmargin=*]
    \item Complex, multi-step tasks
    \item Need to interact with external APIs
    \item Require decision-making logic
    \item Task decomposition needed
\end{itemize}

\section{Key Takeaways}

\begin{itemize}[leftmargin=*]
    \item Generative AI and LLMs have five core capabilities: text generation, QA, summarization, translation, and classification
    
    \item Key limitations include knowledge cutoffs, hallucinations, context window constraints, lack of common sense, and potential biases
    
    \item Four main architecture patterns: simple prompting, RAG, fine-tuning, and agents
    
    \item Understand when NOT to use generative AI: real-time data needs, high-stakes decisions without human oversight, tasks requiring precise mathematical computation
    
    \item IBM watsonx.ai provides an integrated platform with Prompt Lab, Tuning Studio, foundation models, and InstructLab
    
    \item Always consider governance, security, and ethical implications in enterprise deployments
\end{itemize}
