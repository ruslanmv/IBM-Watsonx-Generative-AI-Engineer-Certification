Anatomy of a Production Prompt


<|system|>
You are a senior customer-support agent at ACME Bank.
Answer ONLY from the policy excerpts in <context>.
If the answer is not in context, reply: "I don't know — please contact a human agent."
Tone: formal, concise. Output JSON: { "answer": str, "policy_id": str | null }.
<|context|>
{retrieved_chunks}
<|user|>
{question}
<|assistant|>

Five pieces: role, grounding rule, output format, context, user question.

Zero · Few · Chain-of-Thought

Zero-Shot

No examples. Cheap, fast, great for simple, well-known tasks.

Few-Shot

2–5 examples. Anchors format and tone. Watch context cost.

Chain-of-Thought

"Reason step-by-step." Unlocks logic/math.

Fine-Tuning &
Customization

Where the exam lives. PEFT, LoRA, QLoRA, prompt tuning, InstructLab, Tuning Studio, dataset craft, evaluation.

Hard Prompt vs Soft Prompt

Hard Prompt (text)

Discrete tokens you write. Easy to read, easy to edit. Limited expressivity.

Soft Prompt (vectors)

A learned sequence of continuous embeddings prepended to input. Not human-readable. More expressive per token.

Prompt tuning learns soft prompts only. Granite Tuning Studio default. Smaller artifact than LoRA.

Three Tuning Strategies — Trade-offs

StrategyWhat UpdatesCostQualityArtifact
Full Fine-TuneAll weights$$$$BestFull model (GBs)

LoRA — The Math, Simplified

Original : y = W x LoRA : y = W x + B (A x) └─ trainable └─ rank r (r ≪ d) Init : A ~ Gaussian, B = 0 (so initial output unchanged) Scaling : effective ΔW = α/r · B·A Hyper : r ∈ {4, 8, 16, 32} α ∈ {8, 16, 32} Target : q_proj, v_proj (attention) often enough

PTQ — Post-Training Q

Quantize after training. Fast, simple, mild quality loss.

QAT — Quantization-Aware

Train with simulated quantization. Better quality at INT4.

GPTQ / AWQ

Weight-only INT4 with calibration. Common open-source pipelines.

NF4 (QLoRA)

4-bit "normal float" encoding used by QLoRA.

IBM InstructLab — Detail

🌳 Taxonomy Tree

YAML hierarchy with knowledge and skills branches. Leaves contain seed Q&A.

🧬 SDG (Synthetic Data Gen)

"Teacher" model generates thousands of synthetic examples from each seed leaf.

🏗️ Multi-Phase Training

Phase 1: knowledge. Phase 2: skills. Then alignment (DPO-style).

🤝 LAB Method

Large-scale Alignment for chatBots (Sudalairaj et al. 2024).

Synthetic Data UI — Two Modes

Leverage existing data

Upload a CSV; the generator learns the joint distribution and emits new rows.

Create from schema

Define columns, types, ranges; generate rows from scratch.

Two distribution-fit tests: Kolmogorov–Smirnov (overall CDF) · Anderson–Darling (weights tails heavier).

Differential Privacy Settings

Privacy budget (ε)

Total privacy loss. Smaller = stronger privacy, more noise.

Leakage probability (δ)

Probability the guarantee fails. Typically ≪ 1/N.

Random seed

Reproducibility for audit and testing.

End-to-End RAG with LangChain


from langchain_ibm import WatsonxLLM, WatsonxEmbeddings
from langchain_community.vectorstores import FAISS
from langchain.text_splitter import RecursiveCharacterTextSplitter
from langchain_community.document_loaders import PyPDFLoader
from langchain.chains import RetrievalQA

docs = []
for p in glob.glob("./policies/*.pdf"):
    docs.extend(PyPDFLoader(p).load())

chunks = RecursiveCharacterTextSplitter(chunk_size=512, chunk_overlap=64).split_documents(docs)

emb = WatsonxEmbeddings(model_id="ibm/slate-125m-english-rtrvr-v2", ...)
store = FAISS.from_documents(chunks, emb)

llm = WatsonxLLM(model_id="ibm/granite-3-2-8b-instruct", ...)
qa  = RetrievalQA.from_chain_type(
    llm=llm,
    retriever=store.as_retriever(search_type="mmr", search_kwargs={"k": 4, "fetch_k": 20}),
    return_source_documents=True)

print(qa({"query": "What's the refund policy for enterprise contracts?"}))

Tool-Calling Agent on watsonx


from langchain_ibm import ChatWatsonx
from langchain.agents import create_tool_calling_agent, AgentExecutor
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.tools import tool

@tool
def lookup_account(account_id: str) -> dict:
    "Return account name, tier, status from the internal CRM."
    ...

llm = ChatWatsonx(model_id="ibm/granite-3-2-8b-instruct", ...)
tools = [lookup_account, search_kb, send_email]

prompt = ChatPromptTemplate.from_messages([
    ("system", "You are a senior support agent. Use the tools. Escalate if unsure."),
    ("human",  "{input}"),
    ("placeholder", "{agent_scratchpad}"),
])

agent    = create_tool_calling_agent(llm, tools, prompt)
executor = AgentExecutor(agent=agent, tools=tools, verbose=True)
executor.invoke({"input": "Account 882 reports an outage; draft a reply."})
