The 5 Core Capabilities

1️⃣ Text Generation

Create new content from scratch or complete partial text

2️⃣ Question Answering

Extract or synthesize answers from context or knowledge

3️⃣ Summarization

Condense large information into concise summaries

4️⃣ Translation

Convert between formats, languages, or styles

5️⃣ Classification

Categorize text or detect sentiment

Common Use Cases

  • ✅ Email composition
  • ✅ Report and article writing

Text Generation - Exam Points

When to Use

  • ✓ Creative tasks requiring novelty
  • ✓ Content with acceptable variation
  • ✓ Drafting and ideation

When NOT to Use

  • ✗ Tasks requiring perfect factual accuracy
  • ✗ When specific source citation is needed
  • ✗ High-stakes decisions without review

Capability #2: Question Answering

Type Description Use When
Extractive Q&A Pulls exact text from source documents Compliance-sensitive industries (legal, healthcare)
Abstractive Q&A Synthesizes and generates new answers General customer support, combining multiple sources
Common exam scenario: "Company needs to answer questions about product documentation" → Answer: Retrieval-Augmented Generation (RAG)

Capability #3: Summarization

Real-World Applications

  • 📝 Meeting notes summarization
  • 📄 Document digests and reports
  • 📧 Email thread condensation
  • 📊 Research paper abstracts

Two Approaches

  • Extractive: Selects key sentences from original
  • Abstractive: Generates new summary text
Trade-off: Compression ratio vs information retention. Longer documents may need chunking if they exceed context window

Capabilities #4 & #5

Translation

  • Language translation (EN ↔ ES ↔ FR)
  • Code conversion (Python ↔ JavaScript)
  • Format transformation (JSON ↔ XML)
  • Style transfer (formal ↔ casual)

Classification

  • Sentiment analysis (positive/negative)
  • Intent detection (chatbots)
  • Content moderation (toxicity)
  • Topic categorization (routing)
Both work best with LOW temperature (0.1-0.3) because you want consistent, predictable results

Capability Summary

Capability Temperature Best For Watch Out For
Text Generation 0.7-1.0 Creative content Hallucinations
Question Answering 0.2-0.5 Support, documentation Knowledge cutoff
Summarization 0.3-0.5 Document processing Context length
Translation 0.1-0.3 Format/language conversion Nuance loss
Classification 0.1-0.3 Routing, moderation Edge cases

Key Takeaways for the Exam

  1. Five core capabilities: Generation, Q&A, Summarization, Translation, Classification
  2. Match capability to use case - this is tested extensively
  3. Temperature matters: High for creative, Low for factual
  4. Understand limitations of each capability
  5. Combine capabilities for complex solutions (RAG = Q&A + Retrieval)
Practice matching capabilities to business scenarios - this is a common question pattern on the exam

Practice Question #1

A company wants to automatically route customer support emails to the appropriate department (billing, technical, sales). Which capability is MOST appropriate?

  • A. Text Generation
  • B. Summarization
  • C. Classification
  • D. Translation
Answer: C. Classification
This is an intent classification task - categorizing emails by department. Use low temperature (0.1-0.3) for consistent routing.

Practice Question #2

What temperature setting would be MOST appropriate for generating creative marketing taglines?

  • A. 0.1
  • B. 0.3
  • C. 0.5
  • D. 0.9
Answer: D. 0.9
Creative tasks benefit from higher temperature (0.7-1.0) to generate diverse and novel outputs.
